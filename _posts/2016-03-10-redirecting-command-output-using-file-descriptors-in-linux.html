A terminal shell in linux (bash, zsh) provides three standard file descriptors for any file in the filesystem.

  • STDIN : standard input - file descriptor 0
  • STDOUT : standard output - file descriptor 1
  • STDERR : standard error - file descriptor 2

You can open more file descriptors (such as 3, 4, 5, ...), and you can close them. You can also copy file descriptors. And you can write to them and read from them.

That being said, let's see what we can do with the default file descriptors available for any file:

  • 1. Redirect STDOUT of a command to a file.

    {% highlight bash %} $ ls 1> file_list {% endhighlight %}

    even if we skip to mention file descriptor in our commands, it is defaulted to 1 (STDOUT)

    {% highlight bash %} $ ls > file_list {% endhighlight %}

  • 2. Redirect STDERR of a command to a file.

    {% highlight bash %} $ ls 2> file {% endhighlight %}

  • 3. Redirect both STDOUT and STDERR to a file.

    {% highlight bash %} $ ls -ltra &>file # another way to do it $ ls -ltra >&file {% endhighlight %}

  • 4. Redirect both STDOUT and STDERR to the same destination one after another.

    {% highlight bash %} # STDOUT is redirected first, followed by STDERR $ ls -a >file 2>&1 {% endhighlight %}

    • This can be really handy if you're on console most of the time. Cheers !
